# M3.2 Core API Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Complete the Bilibili API client with cursor-based history pagination, budgeted favorites aggregation, following, comments, search ordering, unified error handling, and lightweight rate limiting.

**Architecture:** Keep a single `BilibiliAPIClient`, but add one shared request helper plus a tiny rate-limit hook. Implement missing endpoints on top of that helper, use cursor-based pagination for history, and put explicit request-budget controls on favorites aggregation so expensive calls cannot silently explode.

**Tech Stack:** Python 3.11+, httpx, pytest, dataclasses
