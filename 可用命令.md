## 🚀 核心命令

### 1. `init` - 系统初始化

**命令格式**：
```bash
openbiliclaw init [OPTIONS]
```

**用途**：首次启动系统，拉取各平台历史数据，生成初始用户画像

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--no-bilibili` | 跳过 B站数据接入 | False |
| `--no-xhs` | 跳过小红书数据接入 | False |
| `--yes-xhs` | 直接启用小红书（跳过 y/n 提问） | False |
| `--no-douyin` | 跳过抖音数据接入 | False |
| `--yes-douyin` | 直接启用抖音 | False |
| `--no-youtube` | 跳过 YouTube 数据接入 | False |
| `--yes-youtube` | 直接启用 YouTube | False |
| `--no-x` | 跳过 X (Twitter) 数据接入 | False |
| `--yes-x` | 直接启用 X | False |
| `--bilibili-favorite-limit` | B站收藏夹拉取数量限制 | 无限制 |
| `--bilibili-history-limit` | B站观看历史拉取数量限制 | 1000 |

**使用示例**：
```bash
# 完整初始化（交互式）
openbiliclaw init

# 只接入 B站和小红书
openbiliclaw init --no-douyin --no-youtube --no-x

# 跳过小红书提示，直接启用
openbiliclaw init --yes-xhs

# 限制 B站数据量（快速初始化）
openbiliclaw init --bilibili-favorite-limit 500 --bilibili-history-limit 300
```

**注意事项**：
- ⚠️ `init` 至少需要保留一个数据来源
- ⚠️ 首次运行会引导配置 LLM provider 和 embedding 服务
- ⚠️ 大量数据拉取可能耗时 10-30 分钟

### 2. `start` - 启动后端服务

**命令格式**：
```bash
openbiliclaw start [OPTIONS]
```

**用途**：启动本地 API 服务，监听指定端口

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--host` | API 监听地址 | 读 config.toml `[api].host` |
| `--port` | API 监听端口 | 读 config.toml `[api].port` |

**使用示例**：
```bash
# 使用配置文件默认值
openbiliclaw start

# 指定监听地址和端口
openbiliclaw start --host 127.0.0.1 --port 8420

# 局域网访问（允许其他设备连接）
openbiliclaw start --host 0.0.0.0 --port 8081
```

**注意事项**：
- ⚠️ 启动前会自动检查数据库健康状态
- ⚠️ 如果配置了密码认证，局域网访问需要登录
- ⚠️ 本机访问（127.0.0.1）免登录

### 3. `serve-api` - 容器友好启动

**命令格式**：
```bash
openbiliclaw serve-api [OPTIONS]
```

**用途**：启动容器友好的 API 服务入口（适合 Docker/Kubernetes）

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--host` | API 监听地址 | 0.0.0.0 |
| `--port` | API 监听端口 | 8420 |

**使用示例**：
```bash
# 容器默认配置
openbiliclaw serve-api

# 自定义端口
openbiliclaw serve-api --host 0.0.0.0 --port 8081
```

**与 `start` 的区别**：
- ✅ `serve-api` 默认监听 `0.0.0.0`（适合容器）
- ✅ `start` 默认读配置文件（适合本地开发）

### 6. `discover` - 触发内容发现

**命令格式**：
```bash
openbiliclaw discover [OPTIONS]
```

**用途**：手动触发内容发现，从指定平台搜索新内容

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--source` | 内容源（bilibili/xiaohongshu/douyin） | bilibili |
| `--strategy` | B站策略过滤（可多次传） | 全部策略 |
| `--limit` | 发现结果条数上限 | 30 |
| `--force` | 小红书强制生产关键词（忽略节流） | False |

**B站策略选项**：
- `search`：关键词搜索
- `trending`：热门视频
- `explore`：探索频道
- `related_chain`：相关视频链

**使用示例**：
```bash
# B站全策略发现
openbiliclaw discover --source bilibili

# 只用搜索和热门策略
openbiliclaw discover --source bilibili --strategy search --strategy trending

# 小红书发现
openbiliclaw discover --source xiaohongshu

# 抖音发现
openbiliclaw discover --source douyin --limit 50

# 强制小红书发现（忽略 4 小时节流）
openbiliclaw discover --source xiaohongshu --force
```

**注意事项**：
- ⚠️ `--strategy` 仅对 B站生效
- ⚠️ 小红书有 4 小时节流机制（防止频繁请求）
- ⚠️ 抖音需要浏览器扩展在线

```bash
阿花：你之前对深度学习和神经网络表现出兴趣...
```

**注意事项**：

### 9. `probe` - 兴趣探针

**命令格式**：
```bash
openbiliclaw probe
```

**用途**：查看并确认/拒绝系统正在试探的兴趣方向

**探针特点**：
- 🔍 **猜测方向**：系统猜测你可能感兴趣的方向
- ✅ **确认机制**：你可以确认或拒绝猜测
- 📊 **置信度**：显示猜测的置信度和确认进度
- 🎯 **细化方向**：帮助系统更精准地推荐

**使用示例**：
```bash
openbiliclaw probe

# 显示活跃的猜测
1. 认知科学 （心理学, 神经科学）
   理由：最近连续浏览高信息密度内容
   确认进度：2/3  置信度：62%

# 输入序号确认（是），序号+n 拒绝（如 1n），或 q 退出
输入序号确认（是），序号+n 拒绝（如 1n），或 q 退出：1
```

**注意事项**：
- ⚠️ 需要先执行 `init` 生成画像
- ⚠️ 确认猜测会加速系统学习
- ⚠️ 拒绝猜测会避免系统误判

## 📊 数据管理命令

### 10. `rebuild-profile` - 重新生成画像

**命令格式**：
```bash
openbiliclaw rebuild-profile [OPTIONS]
```

**用途**：从数据库重新生成灵魂画像（调试用）

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--limit` | 加载的最大事件数 | 5000 |
| `--source` | 只用指定来源（bilibili/xiaohongshu/douyin/youtube） | 全部 |
| `--no-analyze` | 跳过偏好分析，直接重跑画像生成 | False |

**使用场景**：
- ✅ 调整了 LLM prompt 后验证效果
- ✅ 新接入平台后补充旧数据重跑
- ✅ `init` 中途中断后只补跑画像阶段

**使用示例**：
```bash
# 完整重跑（包含偏好分析）
openbiliclaw rebuild-profile --limit 5000

# 只用 B站数据
openbiliclaw rebuild-profile --source bilibili --limit 3000

# 跳过偏好分析（快速重跑）
openbiliclaw rebuild-profile --no-analyze
```

**注意事项**：
- ⚠️ 会消耗 LLM tokens（偏好分析）
- ⚠️ 不需要重新从平台拉取数据

### 11. `profile-consolidate` - 画像整理

**命令格式**：
```bash
openbiliclaw profile-consolidate [OPTIONS]
```

**用途**：用 LLM 整理合并画像里重复的喜欢/讨厌主题

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--apply` | 真正写入合并结果 | False（dry-run） |
| `--revert` | 按 run_id 回滚已应用的整理 | 无 |
| `--migrate-categories` | 迁移一级分类到固定词表 | False |
| `--full` | 全量清理 likes 长尾标签 | False |

**整理流程**：
1. 规则合并（同义词检测）
2. Embedding 聚类（相似主题聚合）
3. LLM 裁决（确认是否合并）
4. 校验执行（写入数据库）

**使用示例**：
```bash
# 查看整理建议（dry-run）
openbiliclaw profile-consolidate

# 应用整理结果
openbiliclaw profile-consolidate --apply

# 回滚某次整理
openbiliclaw profile-consolidate --revert run_20260619_123456

# 迁移一级分类
openbiliclaw profile-consolidate --migrate-categories --apply

# 全量清理 likes 长尾
openbiliclaw profile-consolidate --full --apply
```

**注意事项**：
- ⚠️ 默认 dry-run，先看建议再决定
- ⚠️ `--apply` 会自动备份到 `data/memory/consolidation_runs/`
- ⚠️ 审计记录追加到 `data/memory/soul_changelog.md`

### 12. `fetch-douyin` - 拉取抖音数据

**命令格式**：
```bash
openbiliclaw fetch-douyin [OPTIONS]
```

**用途**：单独触发抖音 bootstrap 拉取（独立于 `init`）

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--wait-seconds` | 等扩展回结果的最大秒数 | 180 |

**采集范围**：
- `dy_post`：发布的视频
- `dy_collect`：收藏的视频
- `dy_like`：点赞的视频
- `dy_follow`：关注的创作者

**前提条件**：
1. ✅ `openbiliclaw start` daemon 在运行
2. ✅ 浏览器扩展已装、service-worker 在线
3. ✅ 浏览器登录了 https://www.douyin.com

**使用示例**：
```bash
# 默认等待 180 秒
openbiliclaw fetch-douyin

# 增加等待时间（抖音滚动较慢）
openbiliclaw fetch-douyin --wait-seconds 240
```

**注意事项**：
- ⚠️ 需要浏览器扩展在线
- ⚠️ 未登录抖音会返回空数据

### 18. `cost` - LLM 调用花费统计

**命令格式**：
```bash
openbiliclaw cost [OPTIONS]
```

**用途**：显示本机 LLM 调用花费统计

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--days` | 统计窗口（1-90 天） | 7 |
| `--by` | 单维度展开（all/day/provider/caller） | all |

**统计维度**：
- `all`：三表全显（按天 + 按 provider/model + 按 caller 模块）
- `day`：按天统计
- `provider`：按 provider/model 统计
- `caller`：按模块统计（discovery/recommendation/soul/api 等）

**使用示例**：
```bash
# 最近 7 天完整统计
openbiliclaw cost --days 7 --by all

# 最近 30 天按天统计
openbiliclaw cost --days 30 --by day

# 按模块统计（排查"钱花在哪一层"）
openbiliclaw cost --days 7 --by caller

# 按 provider 统计
openbiliclaw cost --days 7 --by provider
```

**注意事项**：
- ⚠️ 数据来源：`llm_usage` 表（v0.3.26+）

### 21. `logs-prune` - 日志清理

### 23. `set-password` - 设置访问密码

**命令格式**：
```bash
openbiliclaw set-password [OPTIONS]
```

**用途**：设置/修改局域网访问密码，或关闭门禁/登出所有设备

**核心参数**：
| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--disable` | 关闭密码门禁 | False |
| `--logout-all` | 使所有设备的登录态立即失效 | False |
| `--rotate-secret` | 轮换会话签名密钥（最强撤销） | False |

**使用示例**：
```bash
# 设置密码（交互式）
openbiliclaw set-password

# 关闭密码门禁
openbiliclaw set-password --disable

# 登出所有设备
openbiliclaw set-password --logout-all

# 轮换会话密钥（需重启后端生效）
openbiliclaw set-password --rotate-secret
```

**注意事项**：
- ⚠️ 本机访问（127.0.0.1）免登录
- ⚠️ `--logout-all` 立即生效，不改密码/密钥
- ⚠️ `--rotate-secret` 最强撤销，需重启后端
- ⚠️ 如果设置了环境变量覆盖，config 修改不会生效

### 33. `feedback` - 提交推荐反馈


**使用示例**：
```bash
# 喜欢

openbiliclaw feedback 123 dislike

# 评论
openbiliclaw feedback 123 comment --note "这个视频很有深度"

# 忽略
openbiliclaw feedback 123 dismiss
```

**注意事项**：
- ⚠️ 反馈会影响画像更新
- ⚠️ `comment` 类型必须提供 `--note`

## ⚠️ 重要注意事项

### 1. 前提条件

- ⚠️ 大部分命令需要先执行 `init` 生成画像
- ⚠️ 浏览器扩展相关命令需要扩展在线
- ⚠️ 平台数据拉取需要登录对应平台

### 2. 资源消耗

- ⚠️ `init`、`rebuild-profile`、`profile-consolidate` 会消耗 LLM tokens
- ⚠️ `discover` 可能触发平台 API 频率限制
- ⚠️ 大量数据拉取可能耗时较长

### 3. 配置优先级

- ⚠️ 环境变量优先于 config.toml
- ⚠️ 如果设置了环境变量，某些命令可能无法修改配置

### 4. 数据安全

- ⚠️ Cookie 和凭据存储在本地数据库
- ⚠️ 修改密码/轮换密钥会撤销所有登录态
- ⚠️ 数据库修复前会自动备份
